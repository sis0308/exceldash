🔥 Enable Firebase — Free, One-Time Setup
Allows any employee to login from any city, any laptop, any network — data syncs automatically.
  1. Go to console.firebase.google.com → Create free project
  2. Realtime Database → Create → Test mode
  3. Project Settings → Your apps → Web → Copy config
  4. Create a file called firebase-config.js (see instructions below) → upload separately
  5. Save → Share updated HTML to all employees → Done ✓
💡 Excel sheet data stays local per device. Users, pages, charts, KPIs, slicers sync via Firebase in real time.
🔐 Secure Setup — Create firebase-config.js
1. Create a new file on your computer named firebase-config.js
2. Open it in Notepad and paste this — with YOUR real values:
window.FIREBASE_CONFIG = {
  apiKey: "AIzaSy...your real key...",
  authDomain: "exceldash-sis.firebaseapp.com",
  databaseURL: "https://exceldash-sis-default-rtdb.asia-southeast1.firebasedatabase.app",
  projectId: "exceldash-sis",
  storageBucket: "exceldash-sis.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abcdef"
};
3. Upload firebase-config.js to your GitHub repo (same folder as index.html)
4. Add a .gitignore file in your repo containing just: firebase-config.js
5. Reload the page — green banner will appear ✓
📁 Manual Backup (without Firebase)
Export → send file → user imports on their device.
Last exported: Never
📋 Step-by-step for any PC / Laptop
  1. Admin clicks "Export Dashboard JSON" below — saves a .json file.
  2. Send that file to the user (WhatsApp, email, USB, Google Drive — any way).
  3. User opens this same HTML file in their browser on their PC/laptop.
  4. User logs in with their email & password.
  5. User clicks "Import Dashboard JSON" and picks the file admin sent.
  6. All their permitted pages, charts, KPIs appear immediately ✓
⚠ Import replaces all local data. Users should import only files received from admin.
